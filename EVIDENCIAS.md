# Evidencias
Aquí se encontrarán mis evidencias
## Aprendizajes
1. ¿Qué fue lo más fácil y lo más retador?
Lo más fácil fue comprender todo este módulo con las grabaciones de cada clase pues el profesor con todo el material extra que tiene preparado, siempre nos explico en cada clase a detalle cada paso, cada concepto y procedimiento a seguir para poder aprender sobre la marcha.
Lo más retador fue justamente no poder seguir los pasos sobre la marcha al estar en clase debido a no tener varios dispositivos para poder seguir lo que se estaba realizando, en varios momentos de la clase me perdía, pero gracias a las grabacionesomprendi todo.    
2. ¿Qué etiquetas semánticas usaste y por qué?
Utilicé etiquetas semánticas propias de HTML5 para estructurar el contenido de manera clara y accesible:
<header> para definir el encabezado principal del documento.
<nav> para agrupar los enlaces de navegación.
<main> para delimitar el contenido principal.
<section> para dividir el contenido en bloques temáticos.
<article> cuando trabajé contenido independiente.
<footer> para información final o de cierre.
Las utilicé porque mejoran la organización del documento, para la accesibilidad y tener una estructura adecuada.
3. ¿Cómo organizaste tus commits?
Organicé mis commits de forma progresiva y descriptiva, procurando que cada uno representara un cambio específico o una mejora puntual.
Commit inicial con la estructura base del HTML.
Commit agregando etiquetas semánticas.
Commit incorporando imágenes.
Commit agregando favicon
Commit corrigiendo errores o mejorando estructura.
Intenté mantener mensajes claros y concretos, describiendo qué cambio se realizó y con qué propósito, para facilitar el seguimiento del historial del proyecto.
4. ¿Qué mejorarías en la siguiente iteración?
La planificación previa de la estructura antes de comenzar a codificar.
La organización del repositorio
La redacción de mensajes de commit siguiendo una convención más formal.
